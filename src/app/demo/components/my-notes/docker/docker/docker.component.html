¿Qué es docker?

Docker es una plataforma abierta para desarrollar, enviar y ejecutar aplicaciones.
Docker le permite separar sus aplicaciones de su infraestructura para que usted puede entregar software rápidamente.

Esto facilita la creación de entornos consistentes y reproducibles, lo que ayuda a los desarrolladores y equipos de operaciones a trabajar de manera más eficiente al garantizar que las aplicaciones se ejecuten de la misma manera en diferentes entornos, desde el desarrollo hasta la producción.

¿Qué es un contenedor?

Un contenedor es un entorno aislado para su código. Esto significa que un contenedor no tiene conocimiento de su sistema operativo ni de sus archivos. Se ejecuta en el entorno que le proporciona Docker Desktop. Los contenedores tienen todo lo que su código necesita para ejecutarse, hasta un sistema operativo base.

¿Qué es una imagen?

Una imagen en Docker es un paquete ejecutable que incluye todo lo necesario para ejecutar una aplicación: el código, las bibliotecas, las dependencias, las variables de entorno y la configuración del sistema. Las imágenes se construyen a partir de un archivo llamado Dockerfile, que especifica cómo se debe configurar el entorno. Las imágenes son estáticas y no cambian mientras se ejecutan los contenedores.

¿Qué es un Dockerfile?

Es un archivo de texto que contiene una serie de instrucciones que Docker utiliza para construir una imagen. Las instrucciones en un Dockerfile describen cómo se debe ensamblar la imagen, qué sistema operativo usar, qué software instalar y cómo configurar el entorno para la aplicación.

¿Qué son las Redes en Docker?

Docker proporciona funcionalidades para crear redes virtuales, lo que permite a los contenedores comunicarse entre sí de manera segura y aislada, así como con el mundo exterior.
